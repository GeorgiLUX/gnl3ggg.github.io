How I build websites:

I thing it's important for me to show what my methodology for building websites is, since it is a valuable UX skill.
Work should be completed efficiently, not effectively.
The course I completed for HTML/CSS, courtesy of Ryan Bonhard at Udemy.com, was pretty effective in teaching the student to linearly build modern looking websites, but the point of learning is for thorough understanding not copying a bunch of code and working on a system without fully understanding the how, what and why.

Therefore, I searched for additional knowledge to understand how the most important elements in HTML and CSS work and effect one another, along with some JavaScript and compiled tens of pages worth of notes serving primarily two functions:

  • To quickly recover my knowledge of website building if, in some point in the future, I forget all or most of it.

  • To build websites as quick as systematically possible, thanks to the steps I made to the whole process of website building, the notes being created in a way to compliment the effectiveness of these steps.

My website building steps take into consideration not only my own personal projects, concerned with website building, but the possibility of client existance and meeting their's and their end user's needs and goals:


  1. Receiving the clear idea and goal of the soon-to-be build website

    2. By means of clear communication and empathy, a web-builder, like a UX-er, should be capable of clearly understanding the root of the problem i.e. the need for a website, it's real motivation for existing and end goal.

  2. Creating an early conceptual design by using an existing template, specialised for the main function of the website.

    3. These specialised templates can be downloaded and used with different HTML/CSS frameworks such as - Bootstrap, Foundation, Semantic UI, Purse CSS, etc.
    These framewokrs not only help to not reinvent the wheel when building a website, they provide an intuitive grip on the process and a good level of control over the responsive design.

  3. Gathering the needed material for the website (specialised trademark media content - pictures, videos, logos, icons, etc.).

    4. When constructing a website, materials and knowledge are needed, but without a goal and a story, the indentity of it are non-existent.

  4. Constructing the HMTL/CSS backbone of the pages.

    5. In order to have a fully responsive and user friendly design, no matter the way it is build (grid system, float system, box model, etc.) the backbone of the website is of paramount importance, since proper functioning of the aesthetic look ( CSS styling) is heavily dependant of the existance of a proper foundation.

  5. Inserting the content and implementing the styling.

    6. CSS was created in the mid 90's as a means to describing the presentation of a document, using sets of rules on selectors through a declaration block, by the World Wide Web Consortium. But it's also a way to display the designer's sense of style and aesthetics, a very important trait which gives the website it's feel to the users.

  6. Device and browser optimization.

    7. A website should be made in such a way that it accomodates itself to the users, no matter the device or browser they are using.

  7. Features needed based on user actions, rooted in their motivations.

    8. This step adds a whole array of varied features to websites which I learned at the end of my course. Their function is to better the user experience, the client's needs or both. They include features like - "Back to top" button, varied modals, share buttons, email subscriptions, social meta tags, different forms, smooth scrolling, etc.
